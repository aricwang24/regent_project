# 仓库规则

- 本文件适用于“摄政王-远岬”项目目录及其全部子目录。
- 修改前先阅读相关目录的说明文件，并保持改动范围与任务一致。
- 一切设定核对、大纲规划、正文创作与连续性检查，开始前必须读取 GitHub 仓库 `main` 分支中的当前正式文件；仓库当前内容是唯一工作依据。
- 未经用户在当前任务中明确要求核查或对比历史版本，不得读取、检索或调用项目库 V40、V40以前版本、`archive/` 历史快照、沙盒旧副本及迁移预览。
- 旧版本禁读规则尤其适用于作品主题、价值表达、人物成长、关系含义、结局解释与分卷主题；不得从旧材料提取措辞、补齐空白或影响判断。
- 若用户在当前对话中明确修改设定，应先按用户指示更新仓库，再以更新后的仓库继续工作。
- 未经明确要求，不新增、补写或定稿小说设定、大纲和正文。
- 未经明确要求，不移动、删除或重命名现有文件和目录。
- 文本文件统一使用 UTF-8 编码，保留既有中文命名与 Markdown 结构。
- 日常设定补充、文本修订、新增资料与常规维护默认直接提交到 `main`。
- 涉及批量删除、批量重命名、目录重构或实验性自动化的高风险改动，应使用独立分支和 Pull Request。
- 结构性或正式内容变更应在 `CHANGELOG.md` 中留下简要记录。
