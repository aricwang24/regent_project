# regent-novel 第一版说明

`regent-novel` 是《摄政王·远岬》的项目专用母 Skill。它不保存整套人物卡、机设或故事线，而负责识别任务、最小化加载仓库资料、路由创作模块、执行写前与写后检查，并维护必要的连续性产物。

## 第一版覆盖

- 正史保护与冲突报告；
- 总体故事、分卷、剧情单元、章纲和正文流程；
- 故事池管理；
- 人物、关系边界、车辆装备、现实事实与视觉一致性；
- 总时间轴、人物/车辆状态和章节事实表；
- 番茄连载可读性与出版完整性双重约束；
- 轻量风格卡、场景模式和跨会话交接。

第一版不部署 Hook，不自动改写正文，不安装依赖，不复制第三方小说语料，也不启用多 Agent 强制流程。

## 参考项目与取舍

- `Chinese-WebNovel-Skill`：借鉴短主路由、专项模块下沉、模板与统一完稿审查；未复制其文章语料、类型索引和偏短剧/强反转范例。仓库中未发现独立许可证文件，因此只吸收抽象方法。
- `oh-story-claudecode`：借鉴写前上下文恢复、章/卷流程、追踪文件、会话交接和可降级审查。该项目为 MIT License；本 Skill 仍未复制其 Hook、Agent 部署、自动改写脚本、扫榜或强制爽点系统。
- `NovelClaw`：借鉴人类可编辑的长期记忆、章节摘要和状态检查理念。仓库中未发现独立许可证文件，因此未复制代码；尤其没有采用会自动替换正文和写回避雷库的 `self_check.py` 行为。
- `Awesome-Story-Generation`：借鉴 Plan-and-Write、分层规划、记忆增强、节奏与多维评估的研究方向。它是研究索引且未发现独立许可证文件，未复制论文文本、数据或代码。

## 调用

在 Codex 中使用 `$regent-novel`，并说明任务，例如：

```text
使用 $regent-novel，为第一卷设计一个 2—3 万字的剧情单元，不修改正式设定。
```

```text
使用 $regent-novel，检查这一章的人物关系、车辆空间和时间线连续性。
```
