# 项目产物更新规则

## 写入前

- 确认用户任务授权了相应写入。
- 读目标目录 README 和现有文件。
- 保留 UTF-8、中文命名与既有 Markdown 结构。
- 不移动、删除或重命名现有文件，除非用户明确要求。

## 产物归属

| 产物 | 位置 |
|---|---|
| 正式人物/关系/世界规则 | `01_故事圣经/` |
| 正式车辆与装备 | `02_机设与装备/` |
| 总体、卷、单元、章纲、故事池 | `03_大纲/` |
| 正文 | `04_正文/` |
| 主时间轴、事实表、状态、伏笔 | `05_连续性管理/` |
| 正式视觉资产与说明 | `06_美术资产/` |
| 有来源的现实研究卡 | `07_参考资料/` |
| 草案与过程材料 | `08_创作工作区/` |
| Skill、模板与安全自动化 | `09_工具与自动化/` |

## 写后

- 只更新本次真实变化的台账。
- 新设定先区分“建议”“暂定”“已锁定”；无明确批准不写成已锁定。
- 结构性或正式内容变化写入 `CHANGELOG.md`。
- Git 提交前核对差异，避免把无关文件带入。
- 默认不自动提交或推送；只有用户明确要求或项目任务明确包含 Git 发布时才执行。
