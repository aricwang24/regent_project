# 资料优先级与冲突协议

## 状态词

- **已锁定**：除用户明确修改外不可变。
- **暂定**：可讨论，不可在普通正文任务中静默定稿。
- **待讨论/待锁定**：只能提出选项或记录缺口。
- **需回查**：禁止猜写。
- **已发生**：正文已经发生且进入连续性记录；通常高于旧大纲。

## 冲突裁决

1. 先确认是否真的描述同一事实、同一版本、同一时间点。
2. 按主 `SKILL.md` 的十二级资料优先级排序。
3. 同级冲突时，优先使用专项目录中的无版本号正式文件和较新的明确修订。
4. 已写正文与旧大纲冲突时，若正文已进入连续性记录，保留已发生事实并标记大纲需协调；不得反向改正文。
5. 现实资料与锁定虚构设定冲突时，先判断它是事实错误还是有意架空。只报告，不自动替换小说方案。
6. 无法裁决时输出冲突卡：事实、来源 A、来源 B、影响、建议选项、需要用户决定的问题。

## 禁止的伪裁决

- 用模型记忆填空；
- 将 `archive/`、草稿、聊天摘要或故事池卡提升为正史；
- 为了让当前章节更顺而重写人物边界；
- 为了“更真实”擅改已锁定车辆参数；
- 在大纲目录建立第二套总时间轴。
