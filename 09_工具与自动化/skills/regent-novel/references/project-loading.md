# 项目加载协议

## 入口扫描

1. 定位含 `AGENTS.md`、`00_项目总控/` 和 `01_故事圣经/` 的项目根。
2. 先读根规则，再读 `00_项目总控/设定索引.md`、`正史优先级与文件边界.md` 和目标目录 README。
3. 检查目标文件是否存在，不按旧提示词假定路径。
4. 从索引出发按任务读取，不递归吞入整个仓库。

## 最小读取集合

| 任务 | 必读 | 按需 |
|---|---|---|
| 正式设定 | 总控规则＋目标正式文件 | 迁移映射、归档追溯 |
| 总体故事 | 作品定义＋总体故事线＋主时间轴 | 关系、主线阶段、待定项 |
| 故事池 | 总体故事线＋主时间轴＋相关人物 | 地点池、相关机设 |
| 分卷/剧情单元 | 总体故事线＋主线阶段＋主时间轴 | 人物弧线、故事池 |
| 章纲 | 当前卷/单元＋主时间轴＋前后章纲 | 人物、关系、车辆、研究卡 |
| 正文 | 章纲＋最近 2—3 章＋出场人物＋当前状态 | 车辆、视觉、研究、伏笔 |
| 车辆情节 | 车辆体系上位摘要＋对应专项机设 | 车辆状态、时间轴、相关人物 |
| 关系情节 | 各方人物卡＋核心人物关系＋当前状态 | 人物弧线、最近正文 |
| 现实研究 | 待写情节＋已有研究卡 | 官方/一手外部来源 |
| 美术一致性 | 人物/车辆正式视觉说明＋资产 README | 必要图片、正文场景 |

## 缺失处理

- 缺少章纲：停止正式正文，先补章纲或请求确认。
- 缺少状态表但尚无正文：不是错误，不创建空台账。
- 缺少正式语言基调：使用项目主题和本 Skill 风格卡进行低置信草拟，并明确等待用户审核；不得写成已锁定风格。
- 路径或版本冲突：报告两边证据与影响，不自行移动文件。
