# 会话交接

较长的规划、撰写、审查、研究或仓库维护结束时，使用 [会话交接模板](../templates/session-handoff.md) 留下可恢复记录。交接是工作状态，不是正史来源。

## 必须记录

- 本次目标与实际完成范围。
- 读取过的权威文件和使用的模块。
- 新增或修改的文件，及每项修改的理由。
- 已确认事实、候选判断、未决问题和冲突。
- 使用的外部来源、适用日期与研究卡位置。
- 时间轴、人物、车辆、伏笔等账本是否更新；未更新也说明原因。
- 校验命令或人工检查结果。
- 下一次可以直接执行的 1—3 个动作，以及开始前必须读取的文件。

不要写“已完成研究”“已同步状态”之类无法由文件或命令证明的结论。若本次没有修改正史或连续性账本，应明确写出。
