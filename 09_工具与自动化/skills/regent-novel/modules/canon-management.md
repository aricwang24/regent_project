# 正史管理

仅在用户明确要求新增、修订或整理正式设定时启用。规划章节、润色正文或提出备选方案，都不构成修改正史的授权。

## 读取范围

1. `AGENTS.md`、`00_项目总控/README.md`、`00_项目总控/正史优先级与文件边界.md`。
2. `00_项目总控/设定索引.md` 中与目标条目有关的映射。
3. 目标正式文件及所有直接引用它的摘要、时间轴和状态表。
4. 若涉及机器设定，读取专项机设，不以车辆上位摘要覆盖专项文件。

## 操作规则

- 先列出现状、拟变更内容、受影响文件和潜在冲突，再动笔。
- 保留“正式设定 / 已确认规划 / 候选方案 / 废弃方案 / 存疑”的身份差异。
- 不因现实资料更“合理”而擅自替换已锁定的虚构事实。
- 修改一项权威事实时，同步检查索引、摘要、总时间轴、人物状态、车辆状态和既有正文是否需要明确的衍生更新。
- 结构性或正式设定变化依仓库规则记入 `CHANGELOG.md`。
- 不删除旧决策的必要背景；需要废弃时，注明原因、日期和替代条目。

## 完成门槛

- 新事实有唯一权威归属，不在多个文件里制造平行真相。
- 所有衍生摘要均能回指权威文件。
- 没有把候选内容写成已经发生的剧情。
- 报告所有未解决冲突，不用模糊措辞掩盖差异。
