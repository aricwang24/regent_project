# 正史优先级与文件边界

> 状态：当前正式管理规则。

## 当前工作来源

`aricwang24/regent_project` 的 `main` 分支是本项目唯一的当前工作来源。进行设定核对、大纲规划、正文创作、事实研究或连续性检查前，必须先读取仓库中的相关正式文件。

未经用户在当前任务中明确要求核查或对比历史版本，不得读取、检索或调用以下材料：

- ChatGPT 项目库中的 V40；
- V40以前的任何设定版本；
- 仓库 `archive/` 中的历史快照、旧时间线、修订记录与废案；
- 沙盒旧副本、迁移预览及其他未与当前 `main` 核对一致的副本。

这是一条禁读规则，不是“低优先级参考”规则。作品主题、价值表达、人物成长、关系含义、结局解释与分卷主题尤其不得从旧版本提取、补齐或转述。只有用户在当前任务中明确要求查看历史版本时，才可为该项特定核查读取；核查结果仍不得自动写回现行设定。

当前对话中用户明确作出的新决定优先于既有文件，但应先写回仓库，再作为后续工作的依据。

## 唯一权威来源原则

同一事实只设置一个主要维护位置，其他当前正式文件只保留必要摘要或链接：

| 事实类型 | 主要维护位置 |
|---|---|
| 日期、年龄、事件先后 | `05_连续性管理/时间地点表/总时间轴.md` |
| 人物身份、经历、能力与性格 | `01_故事圣经/人物/` |
| 人物之间的关系变化 | `01_故事圣经/人物关系/核心人物关系.md` |
| 车辆参数、施工、标定与维护 | `02_机设与装备/` 中各车辆专项档案 |
| 车辆归属、叙事功能与能力边界 | `02_机设与装备/车辆体系上位摘要.md` |
| 主线阶段、事故与远征的叙事功能 | `03_大纲/分卷规划/主线阶段与远征.md` |
| 已写章节产生的实时状态 | `05_连续性管理/` 下的状态表；当前尚未建立空表 |
| 外部法规、班次、价格与开放状态 | `07_参考资料/`；写作时重新核查 |

## 冲突时的优先级

1. 当前对话中用户明确作出的新决定；新决定应尽快写回仓库；
2. 《作品定义与主题》对作品气质和核心主题的现行规定；
3. 当前专项目录中的无版本号正式文件；
4. 单独维护的时间轴、人物与关系模块；
5. 当前上位摘要与分卷规划。

车辆专项档案比车辆上位摘要更详细；总时间轴比其他当前摘要中的内嵌时间线更新。被禁读的历史材料不参加优先级排序，也不得用于解决当前文件之间的冲突。

## 状态标记

| 标记 | 含义 |
|---|---|
| 已锁定 | 后续默认不得自行改动；如要修改，必须明确说明是在改设定。 |
| 暂定 | 方向基本确认，但名称、日期、学校、地点或细节仍可调整。 |
| 待讨论 | 已知这是后续必须完成的模块，但当前不替用户决定。 |
| 需回查旧对话 | 当前资料不足以恢复精确原设定，禁止猜写。 |

若某项因为上下文压缩而无法确认，必须标为“需回查旧对话”，不允许为了完整而猜测。历史版本中的锁定标记不具有现行效力，除非它已经存在于当前仓库正式文件中。
